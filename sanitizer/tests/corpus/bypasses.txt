>>> note: Format (parsed by tests/bypass_corpus.rs):
>>> note:   ">>> category: <name>"  sets the category for the vectors that follow
>>> note:   ">>> source: <name>"    sets the provenance label for the vectors that follow
>>> note:   ">>> note: <text>"      human documentation, ignored by the loader
>>> note:   <blank line>            skipped
>>> note:   anything else           a PAYLOAD, taken VERBATIM
>>> note: The ONLY reserved prefix is ">>>", so raw vectors (even ones starting
>>> note: with # or //) paste in unchanged — see SOURCES.md for the re-sync recipe.
>>> note: Every vector MUST be neutralized: no <script>, no on*=, no
>>> note: javascript:/vbscript:/data: in a navigation attribute. A survivor is a
>>> note: sanitizer-contract hole (defense-in-depth framing is in the test header).
>>> note: Inline-style CSS and ARIA-tree vectors are intentionally NOT here.
>>> note: <style>-BLOCK CSS *content* is likewise out of scope (the sanitizer does not
>>> note: CSS-parse it). But <style> *breakout* vectors — ones that try to escape the
>>> note: RAWTEXT element into a live element/handler — ARE included (see awh-style-surface).
